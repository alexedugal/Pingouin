HTML Cheat Sheet

Main Root

Element Description
<html> Represents the root (top-level element) of an HTML document. All other elements must be descendants of this element.

Document Metadata

Metadata contains information about the page, such as styles, scripts, and data to help software (search engines, browsers, etc.) use and render the page. Styles and scripts may be defined in the page or linked to another file.

Element Description
<base> Specifies the base URL for all relative URLs in a document. Only one such element per document.
<head> Contains machine-readable information about the document, such as title, scripts, and style sheets.
<link> Specifies relationships between the current document and an external resource, typically used for linking to CSS files.
<meta> Represents metadata that cannot be represented by other meta-related elements like <base>, <link>, <script>, <style>, and <title>.
<style> Contains CSS styles for the document.
<title> Defines the document's title as shown in the browser's title bar or tab.

Sectioning Root

Element Description
<body> Represents the content of an HTML document. Only one such element per document.

Content Sectioning

Content sectioning elements allow you to organize document content into logical sections.

Element Description
<address> Provides contact information for a person or organization.
<article> Represents a self-contained composition, intended for independent distribution (e.g., blog post).
<aside> Represents content indirectly related to the main content, often styled as a sidebar.
<footer> Represents a footer for its nearest ancestor sectioning element, often containing author info, copyright data, or links.
<header> Represents introductory content, like a logo or navigation aids.
<h1> - <h6> Six levels of section headings, with <h1> as the highest and <h6> as the lowest.
<hgroup> Groups a heading with subheadings or taglines.
<main> Represents the main content of the document, directly related to the central topic.
<nav> Provides navigation links for the page, such as menus or tables of contents.
<section> Represents a standalone section of a document without a more specific semantic element.
<search> Contains form controls or content for performing a search or filter operation.
